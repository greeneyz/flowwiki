Installation

              
                Step 01: Installing the application
                Press "INSTAL" on question 'Do you want to install this application ?'.
                Press "Open". 


                Step 02: Configuring the application
                Press "ALLOW" to give access to photos, media and files.
                Press "ALLOW" to give access to location of device.


                Step 03: Connecting to application
                Enter you email address on 'Welcome Screen' to get the PIN code.
                Press 'Already have code' to enter the PIN code received in the email.
                Press "Send" after entering the PIN.


                Step 04: Understanding the application
                The 'green' color of "Service" and "CAN device" show the connectivity. If it is 'red' then there is no connected device.
                Press 'Dashboard' to see the content.
                Press 'User form' to fill-in and send the user form to backend.
                Press 'Setting -> About' to see the application version, configuration file, ID, and account type.

Configuration

Learn how to configure settings for Scribbler, such as your syntax highlighting preference and the default saving folder location.

Options Value Default
Application configuration Application configuration contains information which are used to customize the application capabilities. true
forcedUpgrade defines how many hours user can continue to use app without application is needed to be updated. true
dataAPIUrl defines URL where data json packages are send. true
dashboardUrls [url, title}defines N iframe URLs which are displayed in application dashboard screen. true
localizationUrl defines location of localization file. true
accountType defines account type. 1= Industrial, 2 = Consumer : capabilities varies for them. 1
devices {btMac, btName} which BT devices are allowed to be connected with application. in MVP list is highlighted with green. NEXT only allowed BT devices can be connected. true
dataFreq defines how often latest values are selected from: GPS, Accelerometer, CAN, BT and AP. - Integer number, milliseconds, minimum allowed value is 1 - Replaces parameters gpsTime, accTime, canFreq, btFreq and apFreq - NOTE that the timestamp for all collected paramter will be set to same. e.g. if dataFreq is 2500, then timestamps will be like 2018-10-10 23:45:40.5, 2018-10-10 23:45:41.0, 2018-10-10 23:45:41.5 etc - The timestamp syncronization makes the grouping for post calculation more efficient. true
gpsAccuracy defines min accuracy for GPS location which is acceptably collected. true
canPgnList if list is empty then all messages are collected. If list has PGNs, then only those messages, found from the PGN list are collected e.g. "" all PGNs are collected e.g. "12FE3401","12FEEF02" only values for these 2 PGNs collected. true
btCount defines how many BT devices are filtered to the list. In order of SSI strenght (highest selected). true
apCount defines how many Access Points are filtered to the list. In order of SSI strentgh (highest selected). true
postTime defines how often collected data is send to backend (sink). true
ts timestamp of the version ? true

User Forms

User forms can be freely modified.
There can be multiple parallel user forms (see defaul as an example).
Note: that the json is very picky of the format. If it's faulty the application will stop working or form is not displayed as you would expect. In worst case you need to re-install the app, since it will keep crashing.
The structure of the form is following:
Each user form can have 4 different kind of data entry types:
Type 1: Tickbox
Type 2: Radiobutton
Type 3: Free text field
Type 4: Pull down selection list
